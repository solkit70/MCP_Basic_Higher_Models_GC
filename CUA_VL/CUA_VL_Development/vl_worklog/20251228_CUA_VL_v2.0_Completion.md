## 💡 Insights & Learnings

### 1. AI 대화형 작성의 효과
**발견**: 라디오 버튼/체크박스 UI를 사용한 구조화된 질문이 매우 효과적
- 사용자가 빠르게 선택 가능
- AI가 필요한 정보를 체계적으로 수집
- 완성된 파일이 즉시 생성됨

**적용**: Step 2 프롬프트에 명시적으로 "선택형 질문으로 물어보면서" 추가

### 2. "다음 단계 안내" 자동화의 중요성
**발견**: 각 단계 완료 후 AI가 다음 단계를 자동으로 제시하면 사용자 경험 향상
- 사용자가 매뉴얼을 다시 찾아볼 필요 없음
- 학습 흐름이 끊기지 않음
- 초보자도 쉽게 따라갈 수 있음

**적용**: Step 3, 4, 5 모든 프롬프트에 "완료되면 다음 단계 안내" 추가

### 3. 기간 검증의 필요성
**발견**: 사용자가 비현실적인 기간을 설정하는 경우 대비 필요
- 너무 짧으면 → 범위 축소 또는 기간 연장 제안
- 너무 길면 → 심화 내용 추가 또는 기간 단축 제안
- 사용자가 최종 결정권을 가지되, AI가 적정성 피드백 제공

**적용**: roadmap_prompt_template.md에 STEP 1 추가, FAQ Q3 업데이트

### 4. CLI/IDE 통합 AI 도구의 중요성
**발견**: 웹 기반 AI만 사용하면 복사/붙여넣기로 시간 낭비
- CLI/IDE 통합 도구는 파일을 직접 읽고 쓸 수 있음
- 워크플로우가 매끄럽게 연결됨
- 효율성이 크게 향상됨

**적용**: FAQ Q1에서 CLI/IDE 통합 도구를 최우선 추천으로 변경

### 5. 배포 버전의 단순성
**발견**: 배포 버전은 최소한의 파일만 포함해야 함
- README.md, GETTING_STARTED.md, templates/ 만으로 충분
- 개발 과정 기록이나 테스트 파일은 혼란만 야기
- 사용자는 3가지 파일/폴더만으로 모든 기능 사용 가능

**적용**: CUA_VL_v2.0 배포 폴더에 필수 요소만 포함
