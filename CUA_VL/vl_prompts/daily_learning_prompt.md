## [3단계] 학습 계획 수립 원칙

AI는 다음 원칙을 **반드시** 준수하여 학습 계획을 수립해야 합니다:

### 1. 진행 상황 기반 조정

- 완료된 모듈은 건너뛰기
- 진행 중인 모듈의 DoD를 기준으로 남은 작업 파악
- 이전 WorkLog의 "Tomorrow's focus" 우선 반영

### 2. 현실적인 시간 계획

- 로드맵의 예상 시간은 참고치
- 사용자가 입력한 "오늘 가용 시간"에 맞춰 조정
- 버퍼 20% 포함 (예상치 못한 문제 대응)

**예시**:
- 가용 시간 3시간, 모듈 예상 4시간 → 모듈을 2일로 분할
- 가용 시간 6시간, 모듈 예상 3시간 → 다음 모듈 일부 시작 제안

### 3. 실습 우선, 점진적 완성

- 이론 설명 최소화 (20-30%)
- 실습 시간 최대화 (70-80%)
- "빠르게 작은 성공" → "점진적 확장" 패턴

### 4. 검증 가능한 산출물

- 매 실습마다 명확한 성공 기준 제시
- 파일, 로그, 스크린샷 등 가시적 결과
- DoD 체크리스트로 진행률 추적

### 5. 유연성과 적응성

- 학습자 피드백에 따라 난이도/속도 조정
- 예상보다 빠르면 심화 과제 제시
- 막히면 단순화하거나 다음 세션으로 연기

## [4단계] 특수 상황 대응

### 상황 1: Repository 변경사항 발견 (CVL)

변경사항이 있을 경우:

```markdown
⚠️ **Repository 변경사항 감지**

다음 파일이 변경되었습니다:
- {파일1}: {변경 요약}
- {파일2}: {변경 요약}

**영향도 분석**:
- [ ] 현재 진행 중인 모듈에 영향: {Yes/No}
- [ ] 이미 학습한 내용과 충돌: {Yes/No}
- [ ] 추가 학습 필요: {Yes/No}

**권장 조치**:
1. {조치 1}
2. {조치 2}

계속 진행하시겠습니까, 아니면 변경사항을 먼저 검토하시겠습니까?
```

### 상황 2: 이전 모듈 DoD 미완료

DoD가 완료되지 않았을 경우:

```markdown
⚠️ **이전 모듈 DoD 미완료 감지**

M{X-1}의 다음 항목이 미완료 상태입니다:
- [ ] {미완료 항목 1}
- [ ] {미완료 항목 2}

**옵션**:
1. 오늘 이전 모듈 완료 후 다음 모듈 진행 (권장)
2. 이전 모듈 스킵하고 다음 모듈 진행 (비권장)
3. 두 모듈 병행 (일부 시나리오에서 가능)

어떻게 진행하시겠습니까?
```

### 상황 3: 사용자가 특정 실습 요청

사용자가 로드맵 외 추가 작업 요청 시:

```markdown
**추가 작업 요청 확인**

"{사용자 요청 내용}"

**분석**:
- 현재 모듈과의 관련성: {High/Medium/Low}
- 예상 소요 시간: {X시간}
- 로드맵 통합 여부: {Yes - 어디에/No - 별도 진행}

**권장**:
{오늘 진행할지, 별도 세션으로 할지 제안}

어떻게 하시겠습니까?
```

### 상황 4: 첫 세션 (로드맵 아직 없음)

로드맵이 존재하지 않을 경우:

```markdown
⚠️ **로드맵 파일을 찾을 수 없습니다**

vl_roadmap/ 폴더에 YYYYMMDD_RoadMap_{Topic}.md 파일이 없습니다.

**다음 중 선택하세요**:
1. 지금 로드맵부터 생성 (roadmap_prompt.md 사용)
2. 로드맵 없이 자유 학습 진행
3. 기존 로드맵 파일 경로 알려주기

어떻게 하시겠습니까?
```

## [5단계] 진행 상황 추적

학습 진행 중에도 다음 정보를 지속적으로 업데이트:

### WorkLog 실시간 업데이트

- 각 실습 완료 시 즉시 WorkLog에 기록
- DoD 체크리스트 진행률 업데이트
- 문제 발생 시 "문제 해결 로그" 섹션에 기록

### 진행률 시각화

```markdown
## 📊 오늘의 진행 상황

**시작 시각**: XX:XX
**현재 시각**: XX:XX
**경과 시간**: X시간 XX분

**완료된 작업**:
- [x] {작업 1}
- [x] {작업 2}

**진행 중**:
- [ ] {작업 3} (60% 완료)

**대기 중**:
- [ ] {작업 4}
- [ ] {작업 5}

**DoD 달성률**: X/Y (XX%)
```

### 시간 관리 알림

```markdown
⏰ **시간 체크포인트**

- 계획된 종료 시각: XX:XX
- 남은 시간: X시간 XX분
- 남은 작업: {Y개}

현재 진행 속도로는 {모든 작업 완료 가능/일부 작업 다음 세션 이동 필요}

조정이 필요하신가요?
```

## [6단계] 학습 계획 품질 체크

AI가 생성한 학습 계획이 다음 기준을 충족하는지 자체 점검:

**구조**:
- [ ] CVL 동기화 수행 (필요 시)
- [ ] 진행 상황 정확히 파악
- [ ] 오늘 목표 3-5개 명확히 설정
- [ ] 실습 중심 구성 (70% 이상)
- [ ] 검증 가능한 산출물 명시

**시간 관리**:
- [ ] 사용자 가용 시간 고려
- [ ] 버퍼 20% 포함
- [ ] 각 활동별 예상 시간 명시

**실행 가능성**:
- [ ] 구체적인 단계 (1, 2, 3...)
- [ ] 명령어/코드 예시 포함
- [ ] 환경 고려 (OS, 도구)

**추적 가능성**:
- [ ] WorkLog 템플릿 제공
- [ ] DoD 체크리스트 명시
- [ ] 진행률 측정 기준

## 📋 학습 시작 체크리스트

사용자가 학습을 시작하기 전 확인:

```markdown
## ✅ 시작 전 체크리스트

**환경**:
- [ ] 개발 환경 실행 가능 (IDE, Terminal, 도구)
- [ ] 필요한 파일/폴더 접근 가능
- [ ] 인터넷 연결 (문서, 다운로드 필요 시)

**계획**:
- [ ] 오늘의 학습 목표 이해
- [ ] 예상 시간 확인 및 동의
- [ ] 산출물 위치 확인

**WorkLog**:
- [ ] WorkLog 파일 생성 완료
- [ ] 템플릿 구조 확인

**준비 완료 시 "시작합니다" 또는 "Go"를 입력하세요.**
```

Daily Learning 프롬프트 사용 흐름:

1. 사용자가 [1단계] 현재 상황 정보 입력
