# CUA_VL 범용 Roadmap 생성 프롬프트

**버전**: 1.0
**생성일**: 2025-12-14
**방법론**: Catch Up AI Vibe Learning (CUA_VL)

---

## 사용 방법

이 프롬프트는 어떤 Topic이든 학습 로드맵을 생성할 수 있는 범용 템플릿입니다.

**사용 절차**:
1. 아래 [1단계] 섹션의 변수들을 학습하려는 Topic에 맞게 작성
2. 전체 내용을 AI에게 전달
3. AI가 CUA_VL 표준에 맞는 로드맵 생성
4. 생성된 로드맵을 `vl_roadmap/YYYYMMDD_RoadMap_{Topic}.md`에 저장

---

## [1단계] Topic 정보 입력 (사용자가 작성)

### 기본 정보

**Topic 제목**:
```
[예시] MCP (Model Context Protocol)
[예시] Docker 컨테이너 기술
[예시] React 프론트엔드 개발
```

**Topic 설명** (2-3문장):
```
[이 기술/프로젝트가 무엇인지, 왜 중요한지, 어디에 사용되는지]

[예시]
MCP(Model Context Protocol)는 AI 애플리케이션이 외부 도구 및 데이터 소스와
표준화된 방식으로 통신할 수 있게 하는 프로토콜입니다.
클라이언트-서버 아키텍처를 통해 다양한 MCP 서버를 통합하고 활용할 수 있습니다.
```

**학습 목적** (최종 목표):
```
[이 학습을 마치면 무엇을 할 수 있게 되는지]

[예시]
- 기존 MCP 서버를 검색하고 내 애플리케이션에 통합
- 커스텀 MCP 서버를 직접 개발하고 배포
- 팀/커뮤니티와 MCP 서버 공유
```

**학습 기간**:
```
[예시] 3일
[예시] 2주
[예시] 1개월
[예시] 5-6주
```

## [2단계] AI에게 요청할 작업

위에 입력한 Topic 정보를 바탕으로 **CUA_VL 방법론**에 맞는 학습 로드맵을 생성해주세요.

---

### 로드맵 생성 요구사항

#### 전체 구조

**학습 기간**: `{입력한 기간}`에 맞춰 조정
- 3일 이하: 3-5개 모듈
- 1-2주: 5-7개 모듈
- 1개월 이상: 7-10개 모듈

**모듈 구성 원칙**:
- 각 모듈은 독립적으로 완료 가능한 단위
- 난이도는 점진적 상승 (Basics → Intermediate → Advanced)
- 마지막 모듈은 Capstone 프로젝트 (통합 실습)

**명명 규칙**:
- 모듈: `M1`, `M2`, `M3`, ...
- 산출물 폴더: `01-{TopicName}/`, `02-{TopicName}/`, ...

---

#### 각 모듈 필수 포함 사항

각 모듈은 다음 구조를 반드시 포함해야 합니다:

**1. 모듈 기본 정보**
```markdown
### MX - {모듈명}

**난이도**: ⭐/⭐⭐/⭐⭐⭐ (1-3)
**예상 시간**: X시간
**산출물 폴더**: ```

**2. 학습 목표** (3-5개)
- 검증 가능하게 작성 ("~을 이해한다" X, "~을 구현할 수 있다" O)
- 체크리스트 형식 `- [ ]`

**3. 주요 개념**
- 핵심 용어 정의 (3-5개)
- 각 개념에 대한 1-2문장 설명
- 오해하기 쉬운 포인트 명시

**4. 실습 과제** (2-3개)
각 실습마다:
- 과제명
- 목적
- 구체적인 단계 (1, 2, 3, ...)
- 예상 소요 시간
- 난이도 표시 (⭐ 1-3개)

**5. 산출물**
- 생성할 폴더 구조
- 필수 파일 목록 (README.md, 코드, 문서 등)
- 권장 하위 폴더 (`concepts/`, `examples/`, `guides/`)

**6. Definition of Done (완료 기준)**
체크리스트 형식으로 5-8개:
```markdown
- [ ] 모든 학습 목표 달성
- [ ] 실습 과제 X개 완료
- [ ] 핵심 명령어/API Y개 실행 성공
- [ ] 산출물 폴더 생성 및 README 작성
- [ ] WorkLog 작성 완료
- [ ] Daily Retrospective 작성
```

**7. Self-Assessment** (자가 평가)
AI 시대에 맞는 평가 기준 (3-5문항):
```markdown
**개념 이해**:
- [ ] 이 기술이 무엇인지 1-2문장으로 설명 가능
- [ ] 왜 사용하는지 예시와 함께 설명 가능

**실무 활용**:
- [ ] AI에게 이 기술을 사용한 작업 요청 가능
- [ ] AI가 생성한 코드의 품질 판단 가능

**문제 해결**:
- [ ] 문제 발생 시 AI에게 디버깅 방향 제시 가능
```

**8. 예상 시간 배분**
```markdown
- 개념 학습: X분
- 실습 1: X분
- 실습 2: X분
- 문서화: X분
- **합계**: X시간 (버퍼 포함)
```

**9. 참조 자료**
- 공식 문서 링크 (필수)
- 튜토리얼/예제 (권장)
- 각 링크마다 1줄 설명

---

#### 실습 설계 원칙 (중요!)

실습 과제를 설계할 때 다음 원칙을 **반드시** 준수하세요:

**1. 실습 우선**
- 이론 설명: 30%
- 실습 시간: 70%
- "개념 설명 → 즉시 실습" 패턴 반복

**2. 점진적 복잡도**
- 실습 1: 간단 (⭐)
- 실습 2: 중간 (⭐⭐)
- 실습 3: 고급 (⭐⭐⭐) - 선택사항

**3. 검증 가능성**
- 모든 실습은 실행 결과로 성공 여부 확인 가능
- 예: "로그 출력", "파일 생성", "API 응답 성공"

**4. AI 시대 학습 범위**
- 인간이 알아야 할 것: 개념, 아키텍처, AI에게 지시하는 방법
- 암기 불필요: 상세 API 문법, 모든 옵션, 내부 구현 디테일

**5. 산출물 중심**
- 매 모듈마다 폴더 생성 (`01-xxx/`, `02-xxx/`)
- "교과서 품질": 다른 학습자가 이것만으로 학습 가능한 수준

**6. 환경 고려**
- 사용자가 입력한 OS/도구에 맞는 명령어 사용
- Windows면 PowerShell, macOS/Linux면 Bash
- 경로 표기도 OS에 맞게 조정

---

#### CUA_VL 방법론 통합

로드맵에 다음 CUA_VL 요소들을 통합하세요:

**1. CVL (Continuous Vibe Learning)**
- Remote Repository를 사용하는 경우:
  - 매 학습 시작 시 동기화 프로세스 포함
  - 변경사항 영향도 평가 기준 제시
  - WorkLog에 동기화 기록 방법 안내

**2. WorkLog 가이드**
- 각 모듈 시작 시 생성할 WorkLog 파일명
- WorkLog에 포함할 필수 섹션
- Daily Retrospective 작성 방법

**3. Retrospective 가이드**
- Module Retrospective: 각 모듈 완료 시
- Topic Retrospective: 전체 완료 시
- 회고에 포함할 항목 명시

**4. 폴더 구조**
```
{Topic}_Project/
├── vl_prompts/
│   └── (이 파일이 여기에 위치)
├── vl_roadmap/
│   └── YYYYMMDD_RoadMap_{Topic}.md (생성할 파일)
├── vl_worklog/
│   ├── YYYYMMDD_M1_{Topic}.md
│   ├── YYYYMMDD_M2_{Topic}.md
│   └── ...
├── 01-{Module1}/
│   ├── README.md
│   ├── concepts/
│   ├── examples/
│   └── guides/
├── 02-{Module2}/
└── ...
```

#### 모듈 예시 (참고용)

아래는 일반적인 학습 Topic의 모듈 구성 예시입니다 (Topic에 따라 조정):

**M1 - 개요 및 핵심 개념**
- Topic이 무엇이고 왜 필요한지
- 주요 용어 및 아키텍처 이해
- Hello World 수준의 첫 실습

**M2 - 환경 설정**
- 개발 환경 구축
- 필수 도구 설치 및 검증
- 프로젝트 스캐폴딩

**M3 - 기초 기능**
- 핵심 기능 3-5개 학습
- 기본 사용 패턴
- 간단한 예제 구현

**M4 - 중급 기능**
- 고급 기능 활용
- 실제 시나리오 적용
- 에러 처리 및 디버깅

**M5 - 통합 및 응용**
- 기존 프로젝트에 통합
- 실무 패턴 적용
- 성능 최적화

**M6 - 배포 및 문서화** (선택)
- 프로덕션 배포
- 문서 작성
- 팀 공유 준비

**MX (마지막) - Capstone**
- 모든 학습 내용 통합
- 완전한 프로젝트 구현
- 최종 산출물 완성

## [3단계] 출력 형식

다음 Markdown 형식으로 로드맵을 생성하고 `vl_roadmap/YYYYMMDD_RoadMap_{Topic}.md`에 저장하세요.

로드맵은 다음 섹션들을 포함해야 합니다:
1. 학습 개요
2. 전체 로드맵 구조 (테이블)
3. 모듈별 상세 계획 (M1, M2, ...)
4. CVL 프로세스 (필요 시)
5. WorkLog 작성 가이드
6. 회고 가이드
7. 폴더 구조
8. 학습 진행 상황 추적 테이블

각 모듈은 앞서 명시한 9가지 필수 항목을 모두 포함해야 합니다.
